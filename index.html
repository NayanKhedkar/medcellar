Welcome to the MedCellar Info Page

Getting Started

To get started with MedCellar you'll need to have a running instance of mongo and you'll need node installed. Both a pretty easy to get up and running and once you get that going it's pretty easy from there.

We decided to leverage grunt to make it easier to reset your environment and to get up and running in the first place. There are two version of MedCellar a weak super insecure version and a development, only slight insecure version. The Insecure version shows a slew of vulnerabilities while the development version implements some security controls to mitigate these vulnerabilities.

The configuration files are located in the application directory under the config folder.

To deploy the weak version:

grunt deployweak

To deploy the development version:

sudo grunt deploy

There are a variety of vulnerabilities inside of MedCellar, but we've implemented a few purposefully that you'll be able to find including:

  • Insecure Direct Object Reference
  • Cross-Site Scripting
  • Authorization Bypass
  • CSRF
  • Header Weaknesses
  • Insecure Random Number Generation
  • SSL Weaknesses
  • Insecure Cookies
  • Local Storage
  • Insecure Hashing
